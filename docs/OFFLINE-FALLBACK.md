# แผนสำรอง — ทำงานต่อได้แม้ DB/เน็ต/Server ล่ม (Offline Resilience)

เป้าหมาย: ถ้าเชื่อมต่อฐานข้อมูลไม่ได้ (เน็ตล่ม/server ล่ม/CORS) **พนักงานยังเปิดดูราคาและทำงานต่อได้** ด้วยข้อมูลล่าสุดที่มี แล้วค่อยซิงก์เมื่อกลับมาออนไลน์

แอปนี้มีของพร้อมอยู่แล้ว (localStorage, Service Worker, Mock adapter, สำรอง .json, ราคาราย DOT ในเครื่อง) → ต่อยอดเป็นระบบสำรองได้เลย

---

## ชั้นที่ 1 — Last-Known-Good Snapshot (หัวใจหลัก)
- ทุกครั้งที่ดึง API สำเร็จ → **บันทึก snapshot ราคาล่าสุดลง localStorage** (`xls2_pricecache` + เวลา)
- เมื่อ server ล่ม → adapter **fallback อัตโนมัติไป snapshot** (ไม่ค้าง/ไม่ขาว)
- แสดงแถบเตือน: "⚠️ ออฟไลน์ — ใช้ราคาล่าสุด ณ 18/06 16:40 (เก่า 2 ชม.)"
- พนักงานยังเห็นราคาทุกตัว · รู้ว่าข้อมูลเก่าแค่ไหน

## ชั้นที่ 2 — Multi-Source Failover (ลำดับสำรอง)
ลองตามลำดับ จนกว่าจะได้:
1. **Server หลัก** (API จริง)
2. **สำรอง LAN** (สำเนา API ในวงเครือข่ายร้าน — เปิด/ปิดไฟยังเข้าได้)
3. **Snapshot ในเครื่อง** (ออฟไลน์เต็มตัว)
→ ตั้ง URL สำรองได้ในหน้า "เชื่อมต่อ DB"

## ชั้นที่ 3 — Service Worker / PWA (เปิดแอปได้แม้ไม่มีเน็ต)
- มี `sw.js` แล้ว → cache app shell (HTML/JS/CSS) → เปิดแอปได้ออฟไลน์ 100%
- ทำเป็น **PWA ติดตั้งลงเครื่อง/มือถือ** ได้ (ไอคอนหน้าจอ เปิดเร็ว)

## ชั้นที่ 4 — Offline Write Queue (Outbox) ✦ ไอเดียล้ำ
- แก้ราคาตอนออฟไลน์ → **ไม่ส่งทันที แต่เข้าคิว** (`xls2_outbox`) + ป้าย "⏳ รอซิงก์ N รายการ"
- พอกลับมาออนไลน์ → **ส่งคิวอัตโนมัติ** + ทำ audit + เคลียร์คิว
- กันเขียนชนกัน: เทียบเวลา/เวอร์ชัน ถ้า server เปลี่ยนระหว่างนั้น → เตือนให้ยืนยัน

## ชั้นที่ 5 — Auto-Retry + สถานะสุขภาพ
- ป้ายแหล่งข้อมูล (เขียว/แดง "ออฟไลน์") มีแล้ว → เพิ่ม **เช็คอัตโนมัติเป็นระยะ** (เช่นทุก 30 วิ) + backoff
- กลับมาออนไลน์เอง → ดึงข้อมูลใหม่ + ป้ายเขียว + เคลียร์ outbox อัตโนมัติ

## ชั้นที่ 6 — สำรองด้วยมือ (มีอยู่แล้ว เสริมให้อัตโนมัติ)
- **Auto-snapshot รายวัน** ลง localStorage (เก็บย้อนหลัง 7 วัน · ย้อนดูได้)
- ปุ่ม **สำรองลงเครื่อง (.json)** + **กู้คืน** — มีแล้ว · ใช้ตอนย้ายเครื่อง/กู้ฉุกเฉิน
- ส่งไฟล์สำรองเข้า Google Drive — มีปุ่มแล้ว

## ไอเดียล้ำเพิ่มเติม (อนาคต)
- **Peer-share ใน LAN:** เครื่องที่มีข้อมูลล่าสุด แชร์ให้เครื่องอื่นในร้าน (WebRTC/LAN) — ไม่ต้องพึ่ง server เลย
- **QR ส่งราคา:** เครื่องหลัก gen QR snapshot → เครื่องอื่นสแกนรับราคา (ออฟไลน์สุดขีด)
- **โหมดฉุกเฉิน:** ปุ่มเดียว "ทำงานออฟไลน์" → ล็อกใช้ snapshot + เปิด outbox + ซ่อนปุ่มที่ต้องต่อเน็ต

---

## ลำดับทำจริง (แนะนำ)
1. **Snapshot + fallback อัตโนมัติ** (ชั้น 1) — คุ้มสุด ทำง่าย ใช้ของที่มี
2. **แถบเตือน + อายุข้อมูล** (ชั้น 1)
3. **Service Worker cache app shell** (ชั้น 3)
4. **Outbox คิวแก้ราคา** (ชั้น 4)
5. **Multi-source / LAN failover** (ชั้น 2)

> เริ่มจากข้อ 1-2 ก่อนได้เลย — แค่นี้พนักงานก็ "ดูราคาต่อได้เสมอ" แม้ server ล่ม

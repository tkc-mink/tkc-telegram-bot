เปิดให้ชิบะคุงอ่านรูปจากนอกวง LAN

ตั้ง Cloudflare Tunnel ให้ AI vision (qwen3.6) ออกเน็ตเป็น https — ปลอดภัย ฟรี ไม่ต้องเปิดพอร์ตเราเตอร์

สำหรับ Mac mini · ทำครั้งเดียว ใช้ได้ตลอด

ปัญหาที่แก้: ตอนนี้ vision อยู่ที่ 192.168.10.38:8080 ซึ่งเป็น IP วงในบ้าน/ร้าน — ออกนอกวง LAN เครื่องอื่นเรียกไม่ได้ และเป็น http จึงโดน mixed-content เวลาเปิดแอปผ่าน https

หลังทำคู่มือนี้: vision จะมี URL แบบ https://vision.โดเมนคุณ เรียกได้จากทุกที่ในโลก และเป็น https → เปิดแอปจากที่ไหนก็อ่านรูปได้ ไม่ติด mixed-content อีก

A ลองแบบเร็ว (ไม่ต้องมีโดเมน)

อยากเทสต์ก่อนว่าใช้ได้ไหม — ได้ URL ชั่วคราวใน 1 นาที

ติดตั้ง cloudflared บน Mac mini (ใช้ Homebrew):

# ถ้ายังไม่มี Homebrew ข้ามได้ ติดตั้งจาก brew.sh ก่อน
brew install cloudflared

เปิด tunnel ชั่วคราวชี้ไปที่ vision (รันบนเครื่องเดียวกับ vision):

cloudflared tunnel --url http://localhost:8080
ถ้า vision อยู่คนละเครื่อง กับที่รัน cloudflared ให้เปลี่ยน localhost เป็น IP วงในของเครื่อง vision เช่น http://192.168.10.38:8080

จะได้ URL หน้าตาแบบนี้โผล่มาในเทอร์มินัล:

https://random-words-1234.trycloudflare.com

เอา URL นั้น + ต่อท้ายด้วย /v1/chat/completions ไปใส่ในชิบะคุง (ดูขั้น C) แล้วลองอ่านรูปได้เลย

ข้อจำกัด: URL ชั่วคราวนี้เปลี่ยนทุกครั้งที่ปิด/เปิดใหม่ และไม่มีรหัสป้องกัน — เหมาะแค่ทดสอบ ใช้งานจริงให้ทำขั้น B

B แบบถาวร (มีโดเมนของตัวเอง)

URL คงที่ ไม่เปลี่ยน + รันเป็นบริการอัตโนมัติ — ต้องมีโดเมนผูกกับ Cloudflare (ฟรี)

ขั้น 1 — ล็อกอิน Cloudflare (เปิดเบราว์เซอร์ให้กดอนุญาต เลือกโดเมนที่จะใช้):

cloudflared tunnel login

ขั้น 2 — สร้าง tunnel ตั้งชื่อว่า vision:

cloudflared tunnel create vision
# จะได้ไฟล์ credential .json + Tunnel ID เก็บไว้

ขั้น 3 — ผูกชื่อโดเมนย่อย (เช่น vision.yourdomain.com):

cloudflared tunnel route dns vision vision.yourdomain.com

ขั้น 4 — สร้างไฟล์ตั้งค่า ที่ ~/.cloudflared/config.yml:

tunnel: vision
credentials-file: /Users/ชื่อผู้ใช้/.cloudflared/TUNNEL-ID.json

ingress:
  - hostname: vision.yourdomain.com
    service: http://localhost:8080   # หรือ IP วงในของเครื่อง vision
  - service: http_status:404

ขั้น 5 — รัน + ติดตั้งเป็นบริการ (เปิดเองทุกครั้งที่บูต Mac):

# ทดสอบรันก่อน
cloudflared tunnel run vision

# ถ้าโอเค ติดตั้งให้รันอัตโนมัติตลอด
sudo cloudflared service install
เสร็จแล้ว! ตอนนี้ https://vision.yourdomain.com/v1/chat/completions เรียกได้จากทุกที่ และเป็น https พร้อมใช้

C ตั้งค่าในชิบะคุง

บอก endpoint ใหม่ให้แชทบอท
  • เปิดแชทชิบะคุง → กดรูปเฟือง ⚙️
  • ช่อง Endpoint ใส่ URL ใหม่ + ต่อท้าย /v1/chat/completions
    เช่น https://vision.yourdomain.com/v1/chat/completions
  • ช่อง โมเดล คงเดิม: Qwen3.6-35B-A3B-NVFP4.gguf
  • ติ๊ก เปิด AI local → กด 🔌 ทดสอบ → ควรขึ้น "✅ เชื่อมต่อ vision สำเร็จ"
  • แนบรูป 📷 ได้เลย — คราวนี้ใช้ได้จากทุกที่ ไม่ต้องอยู่ในวง LAN
โบนัส: เพราะเป็น https แล้ว เปิดแอปจาก claude.ai หรือที่ไหนก็เรียกได้ — ไม่ต้องเปิดผ่าน file:// หรือ http LAN อีกต่อไป

D ความปลอดภัย (สำคัญ)

เพราะตอนนี้ vision ออกเน็ตแล้ว ควรกันคนอื่นแอบใช้

มี 2 ชั้นที่แนะนำให้ทำ (อย่างน้อย 1):

วิธีทำยังไง
API key ที่ llama.cppรัน server ด้วย --api-key รหัสลับ แล้วใส่รหัสเดียวกันในช่อง API key ของชิบะคุง
Cloudflare Accessตั้งใน Cloudflare Zero Trust ให้ต้องล็อกอินอีเมล/รหัสก่อนเข้าถึง URL
อย่าเปิดทิ้งไว้แบบไม่มีรหัส — ใครเดา URL เจอจะเรียกใช้โมเดลคุณได้ฟรี เปลือง GPU และเสี่ยงข้อมูล

E แก้ปัญหาที่เจอบ่อย

อาการสาเหตุ / แก้
